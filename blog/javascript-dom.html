JavaScript Fundamentals

JavaScript and Its relationship to HTML and CSS

If we think about HTML as the bones of a website and CSS as the clothes it wears/how it presents, then we can think of JavaScript as the nervous system of the website rolling out movement and function. The can also be thought of as the verbs in a sentence – the words that create the action.

Control Flow and Loops

Control flow is the sequence or order in which the computer goes through and executes code. Code is generally run from line one to the last line of the file. Exceptions to this are created via functions/operations that change this flow. This is an extremely common occurrence and two such examples would be conditionals and loops.

A real life example might be getting in the car to drive from A to B. so you start the ignition get into gear and remove the handbrake go to take of but you run into an ‘if conditional’ that says that if the petrol indicator is on red you need to detour to a petrol station (which alters the flow of the original journey). After the petrol station you are back on route but you just have to pull over to do full justice to a song you’re listening to. You ‘loop’ through on repeat five times to let the song truly sink in and then your back into the flow again. Finally you are at destination B, into park, pull the handbrake, turn off the ignition and you’re done.

What The DOM?

DOM stands for ‘Document Object Model’ when a document (such as an HTML file) is rendered through a browser the elements are organised into a hierarchy that take the form of a tree structure with one parent per branch– the DOM tree. The topmost node/parent for the page is the document. This object/element representation can act as an interface between the JavaScript and the HTML being represented. This allows JavaScript to make changes to HTML elements and attributes, to change CSS styling, to create new and react to existing events. Leading to dynamic web pages.

Accessing Data, Arrays vs Objects

Objects and arrays are both ways of storing data. Objects store data as key value pairs

const obj = { name: “Banana”, age: 26, phone: 333999}

where name, age, phone are the keys and Banana, 26, and 333999 are the values assigned to those keys. You can search the information/data within an object variable by using the keys. Eg console.log(obj[“name”]) would yield “Banana” Arrays store data as ordered collections of comma separated values. An Array takes the form:

const arr = [“Banana”, “Orange”, “Pear];

Arrays can be accessed by indexes, it is important to note that these start at point 0 rather than one so the first value in the collection will occur at 0. EG console.log(arr[0]) would yield “Banana”

What Are Function and Why Are They Helpful?

Functions as the name implies carry out functions in JavaScript which allows actions and calculations etc to occur. This is extremely helpful as it means that websites can be carry out work and be responsive. Functions take the form, keyword function followed by the name of the function followed by () – the functions parameters followed by {}. Ie

function myFunction() { }

Functions need to be called/ invoked in order to get the function to work. This is done by writing the function name followed by () – houses the functions arguments (values passed to the function when it is invoked.)
