Classes vs IDs

What are they?

HTML and CSS are two languages that are used to create content for websites and to style them respectively. HTML could be thought of as the nouns of a webpage -holds the core meaning and CSS could be viewed as adjectives - are descriptive and alters the nouns. In this analogy it is important to know which "nouns"/HTML elements the "adjectives"/CSS styles are altering. This can be done through the use of selectors. Classes and IDs are both CSS simple selectors, that is, they are attributes used to mark and then change/style groups of HTML elements. CSS selectors conform to a pattern of specificity. For example multiple classes can be assigned to a single element, if these classes conflict the output will default to the last relevant class entry in the CSS information. The class ordering within the HTML element has no effect on which attribute is used. ie

HTML entry

<p class= "pink-text grey-text">HTML entry</p>

CSS entry 

.grey-text {color: gray;} .pink-text {color: pink;}

An ID selector will override a class attribution as there is more importance placed on this demacation.

HTML entry

<p class= "pink-text grey-text" id="special-green">HTML entry</p>

CSS entry 

#special-green {color: green;} .grey-text {color: gray;} .pink-text {color: pink;}

When do I use them and what happens if I use the wrong one?

To recap, you can use CSS classes when you want to repeat styling across multiple elements in a single site/page. Whereas you use an element when you want to apply a specific style to a single element. So what happens if you use id as you would a class - well, it seems that much like using your friends ID sometimes you may get away with it (not advised) but in certain circumstances when it comes to validation, maintenance or adding JS you may run into issues. It is definitly not considered good pracitice to have multple elements sharing IDs!
